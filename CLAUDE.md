# Project Overview
学術論文（英語PDF）の読解効率を向上させるための、翻訳および用語解説機能が統合されたデスクトップ向けのPDFビューア。
「起動の手軽さ」「動作の軽量さ」「データのプライバシー」を重視し、完全にローカル環境で完結するスタンドアロンアプリとして構築する。

# Tech Stack
- **Frontend**: React (TypeScript) + Vite
- **Desktop Framework**: Tauri v2
- **State Persistence**: `@tauri-apps/plugin-store` (セッション・設定の永久保存用)
- **PDF Rendering**: PDF.js
- **Database**: SQLite (Tauriの公式プラグインを利用)
- **LLM API**: Gemini 3.1 Flash-Lite API (※無料枠で利用可能な最新モデル。必要に応じて Gemini 2.5 Flash にフォールバック)

# Core Requirements
## 1. セッションの永久保存と復元（シームレスな再開）
- `@tauri-apps/plugin-store` を使用し、以下のアプリ状態をローカルに永久保存・自動復元すること。
  - ウィンドウのサイズと画面上の位置。
  - 最後に開いていた論文のID（起動時に自動でその論文を開く）。
  - Gemini APIキー（セキュアなローカル保存）。
- アプリを終了し、再度立ち上げた際「前回と全く同じ状態」から待ち時間なく作業を再開できるようにする。

## 2. 基本的なPDFリーダー機能
- ローカルのPDFファイルを選択・アップロードし、アプリ内でレンダリングする（PDF.jsを使用）。
- ページめくり（前へ / 次へ）、総ページ数の表示。
- ズームイン / ズームアウト / ページ幅へのフィット。
- 任意のページ番号へのジャンプ機能。
- **ドキュメントごとの状態記憶**: 各論文の「最後に開いていたページ番号」や「ズーム倍率」をSQLiteに保存し、次回その論文を開いた際に自動適用する。

## 3. テキスト選択とLLM連携（翻訳・解説）
- ユーザーがPDF上のテキストをマウスドラッグで選択したイベントと、その画面上の座標を取得する。
- 選択された英語テキストをGemini APIに送信し、「日本語訳」と「専門用語の解説」を同時に取得する。
- **重要プロンプト要件**: PDF特有の不要な改行コード(`\n`)を無視して元の文章を復元した上で、翻訳と解説を行うようLLMに指示すること。

## 4. UI/UX (ポップアップ表示)
- LLMからのレスポンス（翻訳・解説）を、選択したテキストの近傍（上または下）にポップアップ（ツールチップ）として表示する。
- ポップアップは元のテキストを読む邪魔にならないよう配置する。
- ユーザーがPDFをスクロール・拡大縮小した場合、ポップアップも選択箇所の座標に追従して同じ位置に留まること。
- ポップアップには「閉じる（×）」ボタンを実装し、ユーザーの任意のタイミングで消せるようにする。

## 5. データの永続化（ローカル学習ログ機能）
- 調べた単語、翻訳、解説、およびそのコンテキスト（元の文やページ位置）をローカルのSQLiteに保存し、アプリ内の「履歴・単語帳画面」からいつでも高速に検索・見返しができるようにする。

# Database Schema (SQLite)
ローカル完結型のシンプルな2テーブル構成とする。

## `papers` テーブル（論文メタデータ）
- `id`: INTEGER (Primary Key AUTOINCREMENT)
- `title`: TEXT (論文のタイトルやファイル名)
- `file_path`: TEXT (ローカルファイルの絶対パス)
- `last_opened_page`: INTEGER (最後に開いていたページ)
- `last_zoom_level`: REAL (最後に設定したズーム倍率)
- `uploaded_at`: DATETIME (デフォルト値: 現在日時)

## `highlights` テーブル（翻訳・解説の履歴）
- `id`: INTEGER (Primary Key AUTOINCREMENT)
- `paper_id`: INTEGER (Foreign Key to papers.id)
- `original_text`: TEXT (選択された元の英文)
- `translated_text`: TEXT (LLMによる日本語訳)
- `explanation_text`: TEXT (LLMによる解説文)
- `page_number`: INTEGER (対象テキストが存在するページ番号)
- `created_at`: DATETIME (デフォルト値: 現在日時)

# Development Guidelines for Claude Code
- コンポーネントは機能ごとに細かく分割し、再利用性を高めること（PDF表示部、ポップアップ部、履歴閲覧部など）。
- 状態管理（選択中のテキスト、ポップアップの開閉状態、現在のページなど）は、ReactのContext APIやカスタムフックを用いて綺麗に分離すること。
- スタイリングは一貫性を保つため、Tailwind CSSなどのモダンなCSSフレームワークを推奨する。
- Tauriのネイティブ機能（SQLiteへのアクセス、Storeの読み書き、ファイル選択ダイアログなど）を呼び出す処理は、Frontend側のコンポーネントから直接呼ばず、専用のサービスレイヤー（APIラッパー）を挟んで記述すること。
